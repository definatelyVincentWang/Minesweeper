About this Program

What does this Program do?

This program uses processing 3 and BRC, an external site that creates browser-based controls for processing 3, to make Minesweeper. Minesweeper is a videogame first designed in 1989 and came preinstalled with earlier versions of Windows.

What is Minesweeper

Minesweeper is a puzzle game with the objective of clearing a rectangular board containing hidden "mines" or bombs without detonating any of them. Each tile that is not a mine will have the number of mines surrounding it, and you must use those clues to uncover mines and other unmined tiles. You can win the game by openning all the unmined tiles.

How to Play the Game

To play the game, you have to make sure you have Processing 3 installed. You can go to the processing website and select the correct file for your OS.

Next, download and unzip this file

Open and run the "minesweeper.pde" file, and then go to this website to access the browser based controls

The Controls

To reveal a tile, you have to Left Click the tile. To flag a tile, you have to Right Click the tile.

To change the number of miles or tiles, use the sliders in the browser. You can also use the prebuilt difficulties. You must click the restart button once you change these controls to activate the changes

Also in the BRC website, you will find a stopwatch and a list of Personal Best times for each difficulty. When you win a game, your best time will be saved. Remember, faster is better.

Frequently Asked Questions:

How does it detect if you are clicking a tile or not?

By using the built in built in mousePressed and mouseButton variables, we can determine if the mouse is being pressed and which button is pressed. All we need to do after is to check that the mouse is inside the tile.

How is the Personal High Scores saved?

The personal high scores are stored on your computer in .txt files called Easy, Medium, Hard, and Insane. When you select a difficulty, The program uses the name of the difficulty to determine which file to change. We used the built in loadStrings() function to load everything in a file into a String array and use the built in String.compareTo() function to check which time is less. Finally, we used the createWriter() function to write the best score into the file.

How does the program randomly generate bombs?

Before clicking anything, no bombs are generated. This is to make sure that when the user first presses a tile, it is not a bomb. Once a tile is clicked however, the program will randomly generate an x and y coordinate and as long as those coordinates are not the tile clicked or around the tile clicked, the program will set that tile to a bomb. Once all of the bombs are set, the program will go to each tile and search around the tile to check how many bombs are around it.
